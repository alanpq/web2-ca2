The game is packed with the following base features:

  • Smooth player movement with camera system (wrapper for canvas affine matrix).
  • Input system with virtual button mappings (allowing for multiple keys bound to the same input, etc).
    • Also supports virtual axes, which range from [-1, 1]. An example is the axis vertical, which is 1 when up is held, and -1 when down is held.
  • Infinite grid-based world with procedurally generated rooms and enemies.
  • UI system inspired by imgui.
    • While inspired, I don't actually know how imgui is implemented, so I made it up as I went along.
  • Enemy pathfinding implemented using A* search.

Other technical details

  • This site is a Single Page Application (SPA), hacked into existence by toggling classes.
  • The Player/Enemy classes inherit from Entity, which handles all physics and collisions.

Where is X implemented?

  • Keyboard/mouse functionality is encapsulated by the modules in input/
  • The Debug event handler is implemented in ui/debug.mjs, with many other debug.mjs modules present that are hooked into this event handler.
